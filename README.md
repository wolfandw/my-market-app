# Модуль 2. Спринт 8. Веб-приложение «Витрина интернет-магазина» с использованием Spring Boot на реактивном стеке технологий + кеш Redis + RESTful-сервис платежей и OpenAPI + авторизация

## Описание

Приложение работает во встроенном сервлет-контейнере Netty и доступно по адресу http://localhost:8080/,
использует Embedded In-Memory базу данных H2, использует Redis в качестве кеша товаров, сервер авторизации Keycloack и
является HTTP-клиентом для осуществления запросов в сервис платежей.

Все пользователи делятся на авторизованных и анонимных.
У анонимного пользователя есть доступ только к витрине товаров и к карточке самого товара, при этом он не может
добавлять товары в корзину, переходить в корзину и заказы.
У авторизованного пользователя есть доступ ко всем страницам и элементам управления (кнопкам добавления заказа,
осуществления платежа и т. д.).
Авторизация пользователей (покупателей) осуществляется по логину/паролю. Доступны два предустановленных
пользователя (пароль): admin (admin) и user (user).
Admin имеет право на добавление товаров, редактирование изображений и просмотр всех корзин и заказов.
User имеет право на работу со своей корзиной, своими заказами и оплатой.
В сервере авторизации Keycloack зарегистрированы приложение «Витрина интернет-магазина» как клиент и RESTful-сервис
платежей как сервер ресурсов (авторизация осуществляется по Client Credentials Flow).
Файл настройки клиента: ./realm-export.json

Для работы с приложением необходимо клонировать репозиторий проекта, перейти в каталог проекта, собрать,
протестировать и запустить приложение, а так же сервис платежей, сервер Redis и сервер авторизации Keycloack.
Для комплексного запуска Redis и Keycloack в Docker использовать Docker Compose в корне проекта.

## Начало

- Клонировать репозиторий продукта
- Перейти в каталог продукта

## Сборка, тестирование

- Как собирать:```mvn clean package -DskipTests=true -Dmaven.test.skip=true```
- Как запускать тесты:```mvn test```

## Как запускать

- Запустить в терминале```sudo docker compose up```
  - Проверить наличие образов приложения, сервиса платежей, Keycloack и Redis ```sudo docker image ls```
  - Запустить в терминале```sudo docker container ls``` и убедиться, что контейнеры запущены
  - Остановить контейнеры можно командой```sudo docker compose dowm``` в терминале
  - Запустить заново продукт можно командой```sudo docker compose up``` в терминале
  - При запуске из корневого каталога проекта настройки Keycloack подгружаются автоматически из файла ./realm-export.json
- Собрать приложения
  - Запустить тесты
  - Запустить сервис платежей ```java -jar payment/target/payment-0.0.1-SNAPSHOT.jar```
  - Запустить приложение ```java -jar showcase/target/showcase-0.0.1-SNAPSHOT.jar```
- Перейти по адресу: http://localhost:8080/ на страницу продукта
  - Войти в приложение как admin и заполнить список товаров
