# Модуль 2. Спринт 7. Веб-приложение «Витрина интернет-магазина» с использованием Spring Boot на реактивном стеке технологий + кеш Redis + RESTful-сервис платежей и OpenAPI

## Описание

Приложение работает во встроенном сервлет-контейнере Netty и доступно по адресу http://localhost:8080/,
использует Embedded In-Memory базу данных H2 и использует Redis в качестве кеша товаров и
является HTTP-клиентом для осуществления запросов в сервис платежей.
Для работы с приложением необходимо клонировать репозиторий проекта, перейти в каталог проекта, собрать,
протестировать и запустить приложение, а так же сервис платежей и сервер Redis
Для запуска в Docker можно использовать Docker Compose продукта.

## Начало

- Клонировать репозиторий продукта
- Перейти в каталог продукта

## Сборка, тестирование

- Как собирать:```mvn clean package -DskipTests=true -Dmaven.test.skip=true```
- Как запускать тесты:```mvn test```

## Как запускать вручную

- Собрать приложения
- Запустить тесты
- Запустить Redis, локально или в Docker, например ```sudo docker run --name redis-server -it --rm -p 6379:6379 redis:7.4.2-bookworm```
- Запустить сервис платежей ```java -jar payment/target/payment-0.0.1-SNAPSHOT.jar```
- Запустить приложение ```java -jar showcase/target/showcase-0.0.1-SNAPSHOT.jar```
- Перейти по адресу: http://localhost:8080/ на страницу продукта

## Как собрать образы Docker и запустить Docker Compose

- Собрать и протестировать приложение
  - Запустить в терминале```sudo docker compose up```
  - Проверить наличие образов приложения, сервиса платежей и Redis ```sudo docker image ls```
  - Запустить в терминале```sudo docker container ls``` и убедиться, что контейнеры запущены
  - Остановить контейнеры можно командой```sudo docker compose dowm``` в терминале
  - Запустить заново продукт можно командой```sudo docker compose up``` в терминале
- Открыть браузер и зайти на http://localhost/8080/
